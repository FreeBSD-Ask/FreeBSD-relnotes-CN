## Abstract

The release notes for FreeBSD 15.1-RELEASE contain a summary of the changes made to the operating system on the 15-STABLE development line. This document lists applicable security advisories and errata issued since the last release as well as significant changes to the ecosystem.

## Introduction

This document details the changes since FreeBSD 15.0-RELEASE.

For installation instructions, see the [Installing FreeBSD](https://docs.freebsd.org/en/books/handbook/bsdinstall/) chapter of the FreeBSD Handbook.

For upgrading instructions, see [FreeBSD 15.1-RELEASE Upgrading Instructions](https://www.freebsd.org/releases/15.1R/upgrading).

For information on supported hardware, see the [FreeBSD 15.1-RELEASE Hardware Compatibility List](https://www.freebsd.org/releases/15.1R/hardware).

All readers are encouraged to consult the [FreeBSD 15.1-RELEASE errata](https://www.freebsd.org/releases/15.1R/errata) before installation or upgrading. The errata document contains information discovered late in the release cycle, such as known bugs, security advisories, or documentation corrections.

## Userland

This section covers changes and additions to userland applications, contributed software, and system utilities.

### System Packaging Changes

Changes in this section only affect systems installed using base system packages (sometimes called ​*pkgbase*​), supported in FreeBSD 15.0 and later.

OpenPAM has moved to a new `FreeBSD-pam` package. Systems with the `FreeBSD-set-minimal` or `FreeBSD-set-minimal-jail` package sets installed will install this package automatically, and no user action is required. Systems which do not have either of the minimal sets installed must install the new package if they need to support PAM authentication, e.g. for [login(1)](https://man.freebsd.org/cgi/man.cgi?query=login&sektion=1&format=html) or [sshd(8)](https://man.freebsd.org/cgi/man.cgi?query=sshd&sektion=8&format=html). [95cc7f59b7ce](https://cgit.freebsd.org/src/commit/?id=95cc7f59b7ce).

Zstandard, including the [zstd(1)](https://man.freebsd.org/cgi/man.cgi?query=zstd&sektion=1&format=html) utility, has moved to a new `FreeBSD-zstd` package. Systems with the `FreeBSD-set-minimal` or `FreeBSD-set-minimal-jail` package sets installed will install this package automatically, and no user action is required. Other systems should install this package manually if Zstandard support is required. [8c61751d078e](https://cgit.freebsd.org/src/commit/?id=8c61751d078e).

The `installworld` and `installkernel` targets are now blocked on systems installed from packages to prevent package database inconsistencies. Users wishing to perform source-based upgrades should first unregister the base system packages, as described in [freebsd-base(7)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-base&sektion=7&format=html). [ef82bdc864c5](https://cgit.freebsd.org/src/commit/?id=ef82bdc864c5), [235cb4b4970f](https://cgit.freebsd.org/src/commit/?id=235cb4b4970f).

### Userland Configuration Changes

The `us.intl.acc.kbd` keyboard layout has been installed and available for use with the [vt(4)](https://man.freebsd.org/cgi/man.cgi?query=vt&sektion=4&format=html) console. [34196ce83f9c](https://cgit.freebsd.org/src/commit/?id=34196ce83f9c).

The Lenovo laptop keyboard keymap has been installed and available for use with [vt(4)](https://man.freebsd.org/cgi/man.cgi?query=vt&sektion=4&format=html). [23eaa98d6dc1](https://cgit.freebsd.org/src/commit/?id=23eaa98d6dc1).

The default shell for the root user and the 'freebsd' user in release images is changed from [csh(1)](https://man.freebsd.org/cgi/man.cgi?query=csh&sektion=1&format=html) to [sh(1)](https://man.freebsd.org/cgi/man.cgi?query=sh&sektion=1&format=html). [c8763bd078d8](https://cgit.freebsd.org/src/commit/?id=c8763bd078d8).

rc.subr now supports setting the audit user for services via rc.conf directives, allowing administrators to override the audit user ID when services are started, which prevents unexpected audit log attribution in scenarios like unprivileged users restarting services via sudo. [7edef9117688](https://cgit.freebsd.org/src/commit/?id=7edef9117688). (Sponsored by Modirum MDPay; Klara, Inc.)

### Userland Application Changes

The [find(1)](https://man.freebsd.org/cgi/man.cgi?query=find&sektion=1&format=html) utility now supports -xattr and -xattrname primaries for searching files based on extended attributes. The -xattr primary matches files that have any extended attribute, while -xattrname allows filtering by a specific attribute name, optionally prefixed with a namespace like "user:" or "system:". [5cbb1e05086c](https://cgit.freebsd.org/src/commit/?id=5cbb1e05086c). (Sponsored by Klara, Inc.)

The [newfs(8)](https://man.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&format=html) utility now prevents simultaneous use of GEOM journaling and soft updates. When GEOM journaling is requested, soft updates are not enabled automatically, and explicit requests for both features result in an error. [f758262e66dd](https://cgit.freebsd.org/src/commit/?id=f758262e66dd).

The [tr(1)](https://man.freebsd.org/cgi/man.cgi?query=tr&sektion=1&format=html) utility now correctly handles the CCLASS ([:alpha:]) character class in Unicode mode, fixing case conversion operations for alphabetic characters. [a51ec1cc5a82](https://cgit.freebsd.org/src/commit/?id=a51ec1cc5a82).

The [diff3(1)](https://man.freebsd.org/cgi/man.cgi?query=diff3&sektion=1&format=html) utility’s merge mode is made compatible with GNU diff3, improving interoperability with other systems and tools. [a6f17dd2430b](https://cgit.freebsd.org/src/commit/?id=a6f17dd2430b).

The [pwd(1)](https://man.freebsd.org/cgi/man.cgi?query=pwd&sektion=1&format=html) utility now defaults to the -L (logical) option, following POSIX semantics. Paths containing dot or dot-dot components are rejected in logical mode. [24b95c5ece3d](https://cgit.freebsd.org/src/commit/?id=24b95c5ece3d).

A -E flag has been added to [bectl(8)](https://man.freebsd.org/cgi/man.cgi?query=bectl&sektion=8&format=html) to create an empty boot environment without cloning the currently running environment. This allows users to create a new, independent boot environment for manual installation or configuration. [2b5087e9850b](https://cgit.freebsd.org/src/commit/?id=2b5087e9850b).

The [daemon(8)](https://man.freebsd.org/cgi/man.cgi?query=daemon&sektion=8&format=html) utility now supports specifying the output file mode via a new option, allowing non-root log collectors to access log files and improving compatibility with log rotation tools. [4bc40d5a624e](https://cgit.freebsd.org/src/commit/?id=4bc40d5a624e).

The [newfs(8)](https://man.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&format=html) utility gains a `-u` flag to disable soft updates and soft updates journaling, providing a way to turn off the default soft updates for UFS2 filesystems. [f5a51cfba15c](https://cgit.freebsd.org/src/commit/?id=f5a51cfba15c). (Sponsored by Klara, Inc.; NetApp, Inc.)

The [zfs(8)](https://man.freebsd.org/cgi/man.cgi?query=zfs&sektion=8&format=html) clone command now accepts the `-u` option to prevent automatic mounting of newly created datasets. This allows users to create clones without immediately mounting them, which can be useful for scripting or when preparing datasets for later use. [6ab8f46c6c47](https://cgit.freebsd.org/src/commit/?id=6ab8f46c6c47).

The [zdb(8)](https://man.freebsd.org/cgi/man.cgi?query=zdb&sektion=8&format=html) utility gains a `-O` option for use with `-r` to specify an object ID directly, allowing copying of a file by object ID rather than pathname. [ccb7c82aa15b](https://cgit.freebsd.org/src/commit/?id=ccb7c82aa15b).

[setaudit(8)](https://man.freebsd.org/cgi/man.cgi?query=setaudit&sektion=8&format=html) utility has been added for managing audit policies. [dc88f0003e85](https://cgit.freebsd.org/src/commit/?id=dc88f0003e85). (Sponsored by Modirum MDPay; Klara, Inc.)

The [ipfs(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfs&sektion=8&format=html) utility has been disabled by default and its kernel support is now optional. Users who rely on ipfs must explicitly enable it in the kernel configuration. [91de9b501aa7](https://cgit.freebsd.org/src/commit/?id=91de9b501aa7).

The [sockstat(1)](https://man.freebsd.org/cgi/man.cgi?query=sockstat&sektion=1&format=html) utility now displays UDP-Lite endpoints by default. [c10317b0c455](https://cgit.freebsd.org/src/commit/?id=c10317b0c455).

The [dtrace(1)](https://man.freebsd.org/cgi/man.cgi?query=dtrace&sektion=1&format=html) utility now supports the `ustackdepth` variable on arm64, allowing users to query user-space stack depth in DTrace scripts on that architecture. [00412df3265e](https://cgit.freebsd.org/src/commit/?id=00412df3265e).

DTrace kernel diagnostic messages now use a more intuitive prefix `dtrace`, improving log search consistency. [7781f5e09ffc](https://cgit.freebsd.org/src/commit/?id=7781f5e09ffc).

bsdinstall script now performs pkgbase installation by default, with components specified via COMPONENTS variable, while legacy distset installation remains available if DISTRIBUTIONS is defined. [cb7880bb0900](https://cgit.freebsd.org/src/commit/?id=cb7880bb0900). (Sponsored by ConnectWise)

[cron(8)](https://man.freebsd.org/cgi/man.cgi?query=cron&sektion=8&format=html) now implements full PAM session lifecycle for user jobs, allowing PAM modules to establish credentials and sessions before job execution. PAM environment variables are exported to job processes with user crontab variables taking precedence. A default pam_permit.so session rule is added to /etc/pam.d/cron to enable session support without changing default behavior. [770479c4cea0](https://cgit.freebsd.org/src/commit/?id=770479c4cea0).

### Contributed Software

Certificate authority bundle has been regenerated from NSS 3.123.1, adding new trusted certificates (OISTE Server Root ECC G1, OISTE Server Root RSA G1, SwissSign RSA TLS Root CA 2022, TrustAsia TLS ECC Root CA, TrustAsia TLS RSA Root CA, Certigna Root CA, e-Szigno TLS Root CA 2023, Atos TrustedRoot Root CA ECC G2 2020, Atos TrustedRoot Root CA RSA G2 2020) and removing several certificates. The [pkg(7)](https://man.freebsd.org/cgi/man.cgi?query=pkg&sektion=7&format=html) package is now installed on VM and cloud images that use a packaged base system, enabling base system updates without requiring manual installation of pkg from the ports repository. [b2ba4131b9b0](https://cgit.freebsd.org/src/commit/?id=b2ba4131b9b0). (Sponsored by Amazon)

The EC2 "small" flavour of AMIs no longer runs firstboot_pkgs by default; users who provide a list of packages to install by setting `firstboot_pkgs_list` in EC2 user-data should now also set `firstboot_pkgs_enable="YES"`. [b9ee42d7712b](https://cgit.freebsd.org/src/commit/?id=b9ee42d7712b). (Sponsored by Amazon)

The System Dynamic Tracing (SDT) provider is now enabled on 32-bit powerpc and powerpc64le platforms, allowing users to utilize DTrace probes on these architectures. [769bc6877399](https://cgit.freebsd.org/src/commit/?id=769bc6877399).

The MINIMAL kernel configuration has included the virtio_scsi driver, enabling booting in qemu/kvm virtual machines. [80dec91da334](https://cgit.freebsd.org/src/commit/?id=80dec91da334). (Sponsored by Klara, Inc.; NetApp, Inc.)

A race condition in the powerpc thread switching code that could cause system hangs during boot on multi-CPU power9 pseries guests has been resolved. [a50789d8b89b](https://cgit.freebsd.org/src/commit/?id=a50789d8b89b).

[59ba7278499f](https://cgit.freebsd.org/src/commit/?id=59ba7278499f). (Sponsored by The FreeBSD Foundation)

## Storage

This section covers changes and additions to file systems and other storage subsystems, both local and networked.

### General Storage

The [mt(1)](https://man.freebsd.org/cgi/man.cgi?query=mt&sektion=1&format=html) utility and libmt library add support for LTO-10 and LTO-10P tape density codes and specifications. LTO-10 tape drives are not backward compatible with previous generation LTO tapes. [7fe98ee4d49a](https://cgit.freebsd.org/src/commit/?id=7fe98ee4d49a). (Sponsored by Spectra Logic)

### ZFS

The minimum scrub and resilver times for ZFS have been reduced from multi-second to sub-second values, allowing faster transaction group cycles on modern storage and improving write performance under heavy load. [35ee242abcba](https://cgit.freebsd.org/src/commit/?id=35ee242abcba).

The commit introduces a new tunable, `zfs_arc_evict_batches_limit`, to improve parallel eviction performance in the ZFS ARC. This change reduces lock contention and can significantly increase throughput for certain workloads. [8a79d0968006](https://cgit.freebsd.org/src/commit/?id=8a79d0968006).

The [zpool(8)](https://man.freebsd.org/cgi/man.cgi?query=zpool&sektion=8&format=html) prefetch command now supports BRT (Block Reference Table) metadata, improving performance for block cloning operations and frees of cloned blocks. The `-t` parameter is now optional; when omitted, the command prefetches both DDT and BRT metadata. [41878d57eaf3](https://cgit.freebsd.org/src/commit/?id=41878d57eaf3).

### NFS

The NFS client now properly handles case-insensitive file systems like [msdosfs(4)](https://man.freebsd.org/cgi/man.cgi?query=msdosfs&sektion=4&format=html) when exported via NFS. Name caching has been modified to avoid false cache hits for different case variations of the same file name. [406c98d2ca1d](https://cgit.freebsd.org/src/commit/?id=406c98d2ca1d).

NFSv4 diskless boot now supports configurations using [nfsuserd(8)](https://man.freebsd.org/cgi/man.cgi?query=nfsuserd&sektion=8&format=html) for name and id mappings. The kernel primes the mapping cache with essential entries to avoid deadlocks during boot. Users must set boot.nfsroot.options="nfsv4" and boot.nfsroot.user_domain="<user.domain>" in loader.conf. [692a289922d2](https://cgit.freebsd.org/src/commit/?id=692a289922d2).

Adds internal infrastructure for POSIX draft ACLs as an extension to NFSv4.2, based on an Internet Draft. These changes prepare the NFS client and server for future support but introduce no user-visible semantics changes at this time. Note that due to internal KAPI changes, nfscommon.ko, nfscl.ko, and nfsd.ko must be rebuilt from sources after updating. [20bfa56c514b](https://cgit.freebsd.org/src/commit/?id=20bfa56c514b), [2eddaeb3e505](https://cgit.freebsd.org/src/commit/?id=2eddaeb3e505), [d842c511d84c](https://cgit.freebsd.org/src/commit/?id=d842c511d84c), [dccf41dd1834](https://cgit.freebsd.org/src/commit/?id=dccf41dd1834), [144c1eda3973](https://cgit.freebsd.org/src/commit/?id=144c1eda3973).

The NFSv4 client and server now support the archive attribute (UF_ARCHIVE), which is used by Windows NFSv4.1 clients despite being deprecated in RFC8881. This improves interoperability with Windows NFS clients. [3a64c667f16f](https://cgit.freebsd.org/src/commit/?id=3a64c667f16f).

The support for mounting a diskless root file system via NFSv4 has been added. Requires specific server configuration and client settings in [loader.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=loader.conf&sektion=5&format=html) and [sysctl.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=sysctl.conf&sektion=5&format=html). [2a97b49bc472](https://cgit.freebsd.org/src/commit/?id=2a97b49bc472).

iSCSI TargetName matching on login is now case-insensitive again, restoring RFC 3722 compliance. [379805b29732](https://cgit.freebsd.org/src/commit/?id=379805b29732). (Sponsored by ConnectWise)

## Boot Loader Changes

This section covers the boot loader, boot menu, and other boot-related changes.

### Boot Loader Changes

A shutdown splash screen has been added to the [splash(4)](https://man.freebsd.org/cgi/man.cgi?query=splash&sektion=4&format=html) kernel feature. Users can customize the shutdown splash by providing a PNG image via the `shutdown_splash` directive in [loader.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=loader.conf&sektion=5&format=html). After a kernel panic, the splash screen is no drawn over the debugger output, making panic information visible when ddb(4) is disabled. [0bd9b26d19e2](https://cgit.freebsd.org/src/commit/?id=0bd9b26d19e2), [6a3966f363d3](https://cgit.freebsd.org/src/commit/?id=6a3966f363d3). (Sponsored by Defenso)

The EFI boot loader now uses firmware-provided Blt functions only when using GOP (Graphics Output Protocol), not UGA (Universal Graphics Adapter). This improves compatibility with older MacBook systems where UGA Blt functions were unreliable. [d3902baca506](https://cgit.freebsd.org/src/commit/?id=d3902baca506).

## Networking

This section describes changes that affect networking in FreeBSD.

### General Network

Adds support for masked IP address lookups in [ipfw(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&format=html) tables, allowing non-contiguous prefixes and enabling more CPU-efficient firewall configurations. [a9b93531788c](https://cgit.freebsd.org/src/commit/?id=a9b93531788c). (Sponsored by Yandex LLC)

The [ipfw(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&format=html) firewall now supports masked IP address lookups in tables, allowing non-contiguous prefixes for more efficient firewall rule configurations. This enables CPU-effective firewall setups by masking non-significant bits in lookup keys and table entries. [a9b93531788c](https://cgit.freebsd.org/src/commit/?id=a9b93531788c). (Sponsored by Yandex LLC)

IPv6 packets are now passed to divert sockets, removing a previous limitation that restricted divert sockets to IPv4 only. This enables IPv6 packet handling in divert socket applications. [e5cf3437275f](https://cgit.freebsd.org/src/commit/?id=e5cf3437275f). (Sponsored by OPNsense, Klara, Inc.)

Adds a sysctl/tunable `net.inet.ipf.jail_allowed` to control whether a jail can manage its own ipfilter rules, pools, and settings. The default denies jail access, requiring host system management of jail rules via interface attachment or enabling the sysctl for jail control. [65bc0a1ade72](https://cgit.freebsd.org/src/commit/?id=65bc0a1ade72).

The [ipfw(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&format=html) version 3 compatibility code for FreeBSD 8 and earlier has been removed, as it is obsolete and no longer needed. This cleanup may affect systems still using very old configurations, but it does not change the functionality for current supported releases. [9657c50cdd77](https://cgit.freebsd.org/src/commit/?id=9657c50cdd77). (Sponsored by The FreeBSD Foundation)

The support for RFC 7217 stable IPv6 address generation via a new ifconfig 'stableaddr' flag and associated sysctls has been added. The feature is disabled by default, preserving existing behavior. The [ifconfig(8)](https://man.freebsd.org/cgi/man.cgi?query=ifconfig&sektion=8&format=html) manual page is updated with documentation. [865e0b20ca3c](https://cgit.freebsd.org/src/commit/?id=865e0b20ca3c).

### Wireless Networking

The [iwlwifi(4)](https://man.freebsd.org/cgi/man.cgi?query=iwlwifi&sektion=4&format=html) wireless driver has been updated based on Linux version 7.0. This update includes changes to support newer Intel wireless devices. [f45e34316eb7](https://cgit.freebsd.org/src/commit/?id=f45e34316eb7). (Sponsored by The FreeBSD Foundation)

The [rtw89(4)](https://man.freebsd.org/cgi/man.cgi?query=rtw89&sektion=4&format=html) wireless driver has been updated based on Linux version 7.0, adding support for newer Realtek chipsets and improving stability and functionality. There are still known issues for several chipsets. [7ffebcdb702d](https://cgit.freebsd.org/src/commit/?id=7ffebcdb702d). (Sponsored by The FreeBSD Foundation)

The [rtw88(4)](https://man.freebsd.org/cgi/man.cgi?query=rtw88&sektion=4&format=html) driver has been updated based on Linux version 7.0, incorporating various fixes and improvements for Realtek wireless devices. There are known issues with certain chipsets. [bcdf750def1d](https://cgit.freebsd.org/src/commit/?id=bcdf750def1d). (Sponsored by The FreeBSD Foundation)

Initial support for LinuxKPI-based wireless drivers, notably [iwlwifi(4)](https://man.freebsd.org/cgi/man.cgi?query=iwlwifi&sektion=4&format=html), on RISC-V systems was enabled. While ACPI is not yet fully supported on this architecture, the driver framework has been adapted to allow compilation and operation where possible. Note that the [rtw89(4)](https://man.freebsd.org/cgi/man.cgi?query=rtw89&sektion=4&format=html) driver is temporarily unavailable on RISC-V due to unresolved ACPI dependencies. [7990fa2a819c](https://cgit.freebsd.org/src/commit/?id=7990fa2a819c), [24777428cbc2](https://cgit.freebsd.org/src/commit/?id=24777428cbc2), [d301aadc994f](https://cgit.freebsd.org/src/commit/?id=d301aadc994f), [4a56313c73eb](https://cgit.freebsd.org/src/commit/?id=4a56313c73eb). (Sponsored by The FreeBSD Foundation)

## Hardware Support

### Virtualization Support

Adds a new jail configuration knob allow.vmm_ppt to control PCI passthrough access in [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) jails. The amd64 architecture now supports the Extended Destination ID standard, enabling support for up to 32768 APIC IDs in IO-APIC and MSI without interrupt remapping; in particular, this adds support for EC2 "r8i.96xlarge" instances. [b54646e37a2b](https://cgit.freebsd.org/src/commit/?id=b54646e37a2b). (Sponsored by Amazon)
