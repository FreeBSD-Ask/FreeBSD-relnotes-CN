## 摘要

本文简要说明如何安装 FreeBSD/sparc64 6.3-RELEASE，并着重介绍获取 FreeBSD 发行版的方法。同时提供了一些故障排除和常见问题解答。

## 1 安装 FreeBSD

本节介绍安装全新 FreeBSD 发行版的过程。这些说明着重于获取 FreeBSD 6.3-RELEASE 发行版以及开始安装过程。FreeBSD 手册中的[“安装 FreeBSD”](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/install.html)章节提供了关于安装程序本身的更深入信息，包括带截图的引导式操作 walkthrough。

如果你是从早期版本的 FreeBSD 升级，请参阅[第 3 节](#3-升级-freebsd)获取升级说明。

### 1.1 准备工作

最重要的预安装步骤可能是阅读 FreeBSD 提供的各种说明文档。关于本次 FreeBSD 发行版的文档路线图可在 **README.TXT** 中找到，该文件通常与本文档位于同一位置；大多数文档（如发行说明和硬件兼容列表）也可在安装程序的“文档”菜单中访问。

如果你有互联网连接，还可以从 [FreeBSD 项目网站](http://www.FreeBSD.org/)获取 FreeBSD [FAQ](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/faq/)和[手册](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/)的在线版本。

这套文档看起来可能令人生畏，但花时间阅读它们将节省你大量时间。熟悉可用的资源在安装过程中遇到问题时也会有所帮助。

即使最周密的计划有时也会出错，因此如果遇到问题，请参阅[第 4 节](#4-故障排除)，其中包含有价值的故障排除信息。你还应该在安装前阅读最新的 **ERRATA.TXT** 副本，因为它会提醒你注意特定发行版在发布后报告的任何问题。

>**重要**
>
> 尽管 FreeBSD 尽力防止数据意外丢失，但如果操作失误，仍然极有可能*清除整个磁盘*。在进入最终的 FreeBSD 安装菜单之前，请确保已充分备份所有重要数据。

### 1.2 硬件要求

FreeBSD/UltraSPARC 支持 **HARDWARE.TXT** 中所述的平台。

你需要为 FreeBSD/sparc64 准备一块专用磁盘。目前无法与其他操作系统共享磁盘。

如果你不熟悉为 FreeBSD 配置硬件，务必阅读 **HARDWARE.TXT** 文件；其中包含 FreeBSD 支持哪些硬件的重要信息。

### 1.3 软盘镜像说明

FreeBSD/sparc64 不支持基于软盘的安装。

### 1.4 从 CDROM 或互联网安装 FreeBSD

大多数 sparc64 系统设置为从磁盘自动启动。要安装 FreeBSD，需要通过网络或从 CDROM 启动，这需要你进入 PROM（OpenFirmware）。

为此，重启系统并等待启动消息出现。具体取决于型号，但应该类似于：

```sh
Sun Blade 100 (UltraSPARC-IIe), Keyboard Present
Copyright 1998-2001 Sun Microsystems, Inc.  All rights reserved.
OpenBoot 4.2, 128 MB memory installed, Serial #51090132.
Ethernet address 0:3:ba:b:92:d4, Host ID: 830b92d4.
```

如果系统此时从磁盘启动，需要按键盘上的 **L1**+**A** 或 **Stop**+**A**，或通过串口控制台发送 `BREAK`（例如在 [tip(1)](http://www.freebsd.org/cgi/man.cgi?query=tip&sektion=1&manpath=FreeBSD+6.3-RELEASE) 或 [cu(1)](http://www.freebsd.org/cgi/man.cgi?query=cu&sektion=1&manpath=FreeBSD+6.3-RELEASE) 中使用 `~#`）以进入 PROM 提示符。它看起来像这样：

- 单 CPU 系统使用 `ok` 提示符
- SMP 系统使用 `ok {0}` 提示符（数字表示活动 CPU 的编号）

此时，将 CDROM 放入驱动器，并在 PROM 提示符下输入 `boot cdrom`。

### 1.5 各种安装类型的详细说明

一旦你以某种方式进入初始安装屏幕，应该能够按照各种菜单提示进行操作。如果你以前从未使用过 FreeBSD 安装程序，也建议阅读“文档”子菜单中的一些文档以及第一个菜单上的通用“Usage”说明。

>**注意**
>
> 如果在某个屏幕卡住，按 **F1** 键可获取该特定章节的在线文档。

如果你从未安装过 FreeBSD，建议使用“Standard”安装模式，因为它确保你会经过所有重要的检查清单项。如果你对 FreeBSD 安装过程非常熟悉并且*确切*知道要做什么，可使用“Express”或“Custom”安装选项。如果你要升级现有系统，使用“Upgrade”选项。

FreeBSD 安装程序支持直接使用软盘、DOS、磁带、CDROM、FTP、NFS 和 UFS 分区作为安装媒体；以下列出每种媒体类型的安装提示。

#### 1.5.1 从网络 CDROM 安装

如果你只想从本地 CDROM 驱动器安装，请参阅[第 1.4 节](#14-从-cdrom-或互联网安装-freebsd)。如果你的系统没有 CDROM 驱动器，想使用另一台你有网络连接的机器的 CDROM 驱动器中的 FreeBSD 发行版 CD，有几种方式：

- 如果你可以直接从某台 FreeBSD 机器的 CDROM 驱动器通过 FTP 安装 FreeBSD，操作很简单：确保 FTP 服务器正在运行，然后使用 [vipw(8)](http://www.freebsd.org/cgi/man.cgi?query=vipw&sektion=8&manpath=FreeBSD+6.3-RELEASE) 命令在密码文件中添加以下行：

```sh
ftp:*:99:99::0:0:FTP:/cdrom:/sbin/nologin
```

在运行安装的机器上，进入 Options 菜单并将 Release Name 设置为 `any`。然后选择 `FTP` 作为 Media 类型，并在 ftp 站点菜单中选择“URL”后输入 `ftp://machine`。

>**警告**
>
> 这可能允许本地网络（或互联网）上的任何人对此机器进行“匿名 FTP”连接，这可能不是所希望的。

- 如果你更愿意使用 NFS 直接将 CDROM 导出给你要安装的机器，需要先在 **/etc/exports** 文件中添加条目。以下示例允许 `ziggy.foo.com` 在安装期间通过 NFS 直接挂载 CDROM：

```sh
/cdrom          -ro             ziggy.foo.com
```

有 CDROM 的机器还必须配置为 NFS 服务器。如果你不确定如何操作，那么 NFS 安装可能不是最佳选择，除非你愿意阅读 [rc.conf(5)](http://www.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&manpath=FreeBSD+6.3-RELEASE) 并进行适当配置。

#### 1.5.2 从软盘安装

FreeBSD/sparc64 不支持从软盘安装。

#### 1.5.3 从 DOS 分区安装

FreeBSD/sparc64 不支持从 DOS 分区安装，因为 sparc64 平台不使用 DOS 文件系统。

#### 1.5.4 从 QIC/SCSI 磁带安装

从磁带安装时，安装程序期望文件简单地以 tar 格式存储在磁带上。使用 [tar(1)](http://www.freebsd.org/cgi/man.cgi?query=tar&sektion=1&manpath=FreeBSD+6.3-RELEASE) 将发行版文件写入磁带：

```sh
# cd /where/you/have/your/dists
# tar cvf /dev/sa0 dist1 .. dist2
```

进行安装时，还应确保在某个临时目录中留出足够空间以容纳所创建磁带的*全部*内容。由于磁带的非随机访问特性，这种安装方法需要相当多的临时存储空间。

>**注意**
>
> 进行安装时，必须在启动*之前*将磁带放入驱动器。

#### 1.5.5 通过网络使用 FTP 或 NFS 安装

通过网络加载安装内容可以使用三种连接类型之一：串口、并口或以太网。

##### 1.5.5.1 串口

SLIP 支持相当原始，主要限于硬连线链路。该链路必须是硬连线的，因为 SLIP 安装目前不提供拨号功能。如果需要使用调制解调器拨号，应使用 PPP 工具。

如果使用 PPP，确保手头有互联网服务提供商的 IP 地址和 DNS 信息。

##### 1.5.5.2 并口

如果有到另一台 FreeBSD 或 Linux 机器的硬连线连接，还可以考虑通过“laplink”式并口电缆安装。并口的数据传输率远高于串行线路通常可能的速率（最高 50k/sec）。

>**重要**
>
> 如果使用 Linux 机器而非 FreeBSD 机器作为 PLIP 对端，还需在 TCP/IP 设置屏幕的“ifconfig 附加选项”字段中指定 `link0`，以兼容 Linux 略有差异的 PLIP 协议。

##### 1.5.5.3 以太网

FreeBSD 支持大多数常见的以太网卡；支持的网卡列表作为 FreeBSD 硬件说明的一部分提供（参见 **HARDWARE.TXT**）。

你还需要知道你在网络上的 IP 地址、子网的 `netmask` 值以及你的机器名称。系统管理员可以告诉你哪些值适合你的特定网络设置。

如果你不知道这些问题的答案，那么在尝试此类安装之前应该先与系统管理员讨论。

一旦某种网络连接正常工作，安装就可以通过 NFS 或 FTP 继续。

##### 1.5.5.4 NFS 安装提示

NFS 安装相当直接：只需将你想要的 FreeBSD 发行版文件复制到某个服务器上，然后将 NFS 媒体选择指向它。

如果此服务器仅支持“特权端口”访问，你可能需要在 Options 菜单中设置此选项才能继续安装。

为使 NFS 安装正常工作，服务器还必须支持“子目录挂载”。在 FreeBSD 的 **/etc/exports** 文件中，这由 `-alldirs` 选项控制。

##### 1.5.5.5 FTP 安装提示

可以从任何包含相当新的 FreeBSD 版本的镜像站点进行 FTP 安装。安装期间的 FTP 站点菜单提供了几乎适用于世界上任何位置的合理选择列表。

如果你要从此菜单中未列出的其他 FTP 站点安装，还可以通过在该菜单中选择“URL”选项来指定自己的 URL。URL 可以包含主机名或 IP 地址。

有三种 FTP 安装模式可供使用：

- **FTP**：使用标准的“Active”模式进行传输。
- **FTP Passive**：设置 FTP“Passive”模式，阻止服务器向客户端打开连接。适合需要通过防火墙的用户。
- **通过 HTTP 代理的 FTP**：使用 HTTP 连接到代理进行所有 FTP 操作。允许通过不允许 FTP 但提供 HTTP 代理的防火墙。

## 2 发行版格式

FreeBSD 发行版以目录结构组织。顶级目录包含各种子目录，每个子目录对应一个发行版组件。主要组件包括：

- **bin**：基本系统二进制文件和库
- **src**：系统源代码
- **ports**：Ports 树
- **doc**：文档
- **manpages**：手册页
- **proflibs**：性能分析库
- **dict**：字典文件
- **info**：信息文件
- **catpages**：预格式化的手册页
- **kernels**：预编译的内核
- **games**：游戏

每个发行版组件都包含一个 `.inf` 文件，描述了该组件的结构和所需的磁盘数。安装程序使用这些信息来正确获取和拼接发行版文件。

## 3 升级 FreeBSD

### 3.1 概述

FreeBSD 6.3-RELEASE 的源代码升级仅支持从 FreeBSD 5.3-RELEASE 或更高版本进行。较旧系统的用户若要升级到 6.3-RELEASE，需先更新到 FreeBSD 5.3 或更新版本，然后再升级到 FreeBSD 6.3-RELEASE。

>**重要**
>
> 升级 FreeBSD 前，请务必先备份**所有**数据和配置文件。

### 3.2 升级步骤

1. **备份**：在开始升级过程之前，请备份所有重要数据和配置文件。
2. **挂载文件系统**：如果从 CDROM 启动，挂载现有的根文件系统。
3. **选择发行版**：选择“Upgrade”安装选项，然后选择要安装的发行版组件。
4. **安装后配置**：升级完成后，运行 `mergemaster` 工具合并配置文件更改。
5. **源代码升级**：如果要从源代码升级，请使用 CVS 或 SVN 获取源代码，然后按照标准源代码升级过程进行。

### 3.3 从源代码升级

对于 CVS，用于更新源代码的分支标签为 `RELENG_6_3`。对于 SVN，请使用 `releng/6.3`。

从源代码升级的过程请参阅 FreeBSD 手册中的[同步源代码](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/synching.html)和[构建世界](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/makeworld.html)章节。

## 4 故障排除

### 4.1 修复现有安装

如果你的 FreeBSD 系统由于某种原因无法启动，可以使用安装 CDROM 启动到修复模式。在 PROM 提示符下输入：

```sh
ok boot cdrom
```

进入单用户模式后，可以挂载根文件系统并修复问题。要在启动时进入单用户模式，使用：

```sh
ok boot -s
```

### 4.2 常见安装问题

#### 4.2.1 我的系统在启动时探测硬件时挂起。

可以通过在 PROM 提示符下设置 `hint.acpi.0.disabled` 来禁用 ACPI：

```sh
set hint.acpi.0.disabled="1"
```

每次系统启动时此设置都会重置，因此需要将 `hint.acpi.0.disabled="1"` 添加到 **/boot/loader.conf** 文件中。

#### 4.2.4 安装 FreeBSD 后首次从硬盘启动时，内核加载并探测硬件，但停在以下消息处：

```sh
changing root device to ad1s1a panic: cannot mount root
```

如果启动磁盘不是系统中的第一个磁盘，存在一个长期存在的问题。需要告诉 FreeBSD 根文件系统的位置。通过指定磁盘类型和 FreeBSD 磁盘编号来完成。
