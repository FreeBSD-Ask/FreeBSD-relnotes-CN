### 2.2 在 Alpha 上运行 FreeBSD 的基本要求

显然，你需要一台 FreeBSD 了解的 Alpha 机器。Alpha 机器*不像* PC。各种核心逻辑芯片组和主板设计之间存在相当大的差异。这意味着内核在运行之前需要了解特定机器的详细信息。向未知硬件抛出某个 **GENERIC** 内核几乎肯定会惨败。

对于要考虑用于 FreeBSD 的机器，请确保已安装 SRM 控制台固件。或者至少确保特定机器类型有 SRM 控制台固件可用。如果 FreeBSD 当前不支持你的机器类型，假设 SRM 可用，这很可能在某个时候会改变。当 SRM 控制台固件不可用时，一切都不确定。

带有 ARC 或 AlphaBIOS 控制台固件的机器是为 WindowsNT 设计的。有些机器在系统 ROM 中有 SRM 控制台固件可用，你只需选择（通过 ARC 或 AlphaBIOS 菜单）。在其他情况下，你需要用 SRM 代码重新刷新 ROM。请查看 **http://ftp.digital.com/pub/DEC/Alpha/firmware** 了解特定系统可用的固件。无论如何：没有 SRM 就意味着*没有* FreeBSD（NetBSD、OpenBSD、Tru64 Unix 或 OpenVMS 也是如此）。随着 WindowsNT/alpha 的消亡，许多以前的 NT 机器在二手市场上出售。当它们仅支持 NT 时，几乎没有或没有折旧价值。所以，如果价格看起来太便宜，请保持怀疑。

如果你在选择 SRM 时遇到问题（换句话说，系统在你选择 SRM 控制台后回退到 AlphaBIOS），是时候检查 CMOS 时钟/参数 RAM 的电池是否耗尽了。如果是，更换电池很可能会解决问题。

已知的非 SRM 机器有：

- Digital XL 系列
- Digital XLT 系列
- Samsung PC164UX（“Ruffian”）
- Samsung 164B

有 SRM 但不受 FreeBSD 支持的机器有：

- DECpc 150（“Jensen”）
- DEC 2000/300（“Jensen”）
- DEC 2000/500（“Culzean”）
- AXPvme 系列（“Medulla”）

让事情更复杂的是：Digital 曾经有所谓的“白箱”Alpha 机器（ destined 为 NT-only）和“蓝箱”Alpha 机器（destined 为 OpenVMS 和 Digital Unix）。这些名称基于机箱颜色，分别为“FrostWhite”和“TopGunBlue”。虽然你可以在白箱上安装 SRM 控制台固件，但 OpenVMS 和 Digital Unix 会拒绝在它们上启动。FreeBSD 在 post-4.0-RELEASE 中可以在白箱和蓝箱两种变体上运行。在有人问之前：白色的机器有一个相当不同的（即更便宜的）Digital 价格标签。

作为 SRM 的一部分，你会得到所谓的 OSF/1 PAL 代码（OSF/1 是 Digital 在 Alpha 上的 UNIX 产品的最初名称）。PAL 代码可以被视为硬件和操作系统之间的软件抽象层。它使用普通 CPU 指令加上少量专用于 PAL 使用的特权指令。PAL 不是微代码。ARC 控制台固件包含不同的 PAL 代码，面向 WinNT，完全不适合 FreeBSD 使用（更一般地说：Unix 或 OpenVMS）。在有人问之前：Linux/alpha 自带 PAL 代码，允许它在 ARC 和 AlphaBIOS 上启动。*BSD 阵营认为这不是一个好主意，原因有多种。我在这里不想详细讨论。如果你对血腥细节感兴趣，请搜索 FreeBSD 和 NetBSD 网站。

前面还有一个陷阱：你需要一个 SRM 控制台固件识别的磁盘适配器才能从磁盘启动。遗憾的是，SRM 接受什么作为启动适配器高度依赖于系统和 SRM 版本。对于较旧的基于 PCI 的机器，这意味着你需要基于 NCR/Symbios 53C810 的适配器，或基于 Qlogic 1020/1040 的适配器。有些机器在主板上嵌入了 SCSI 芯片。较新的机器设计和 SRM 版本能够与更现代的 SCSI 芯片/适配器一起工作。请查看下面的机器特定信息。请注意，本讨论的其余部分仅涉及 Symbios 芯片，这意在包括仍印有 NCR 标志的较旧芯片。Symbios 在某个时候收购了 NCR。

这个问题可能会困扰那些机器最初是 WindowsNT 机器的用户。ARC 或 AlphaBIOS 知道*其他*类型的适配器可以从中启动，而 SRM 则不同。例如，你可以用 ARC/AlphaBios 从 Adaptec 2940UW 启动，但（通常）不能用 SRM。一些较新的机器类型引入了 Adaptec 启动支持。详情请查阅机器特定章节。

大多数不能启动的适配器对于仅数据磁盘工作正常。SRM 和 ARC 之间的差异也可能在某些（以前的 WindowsNT）系统中为你带来预装的 IDE CDROM 和硬盘。存在可以从 IDE 磁盘和 CDROM 启动的 SRM 版本（取决于机器类型）。详情请查看机器特定章节。

为了可启动，根分区（分区 a）必须位于磁盘驱动器的偏移 0 处。这意味着你必须使用安装程序的分区菜单，并从将分区 a 分配在偏移 0 处的根分区开始。然后按你的喜好布局其余分区。如果你不遵守此规则，安装将顺利进行，但系统将无法从新安装的磁盘启动。请注意，在 Alpha 上运行的 FreeBSD 不使用 fdisk，磁盘使用 disklabel 直接标记。

如果你没有/不想要本地磁盘驱动器，你可以通过以太网启动。这需要一个 SRM 控制台识别的以太网适配器/芯片。一般来说，这归结为基于 21040 或 21142 或 21143 的以太网接口。较旧的机器或 SRM 版本可能无法识别 21142 / 21143 快速以太网芯片，那么你只能使用 10Mbit 以太网进行网络启动。基于所述芯片的非 DEC 卡通常可以工作（但不保证）。请注意，Intel 在收购 Digital Semiconductor 时接管了 21x4x 芯片。所以你近来可能会在它们上看到 Intel 标志。最近的机器设计有对 Intel 8255x 以太网芯片的 SRM 支持。

Alpha 机器可以使用 SRM 在图形控制台或串行控制台上运行。如果需要，ARC 也可以在串行控制台上运行。具有 8 位控制的 VT100 仿真至少应该允许你在不先安装显卡的情况下从 ARC/AlphaBIOS 切换到 SRM 模式。

如果你想在没显示器/显卡的情况下运行 Alpha 机器，只需不连接键盘/鼠标到机器。相反，将串行终端[模拟器]连接到串行端口 #1。SRM 会以 9600N81 与你通信。这对于调试目的也非常实用。注意：一些/大多数（？）SRM 也会在串行端口 #2 上为你提供控制台提示符。但是，启动内核会在串行端口 #1 上显示启动消息，并也会将控制台放在那里。*这可能非常令人困惑。*

大多数基于 PCI 的 Alpha 可以使用普通 PC 型 VGA 卡。SRM 包含足够的智能使其工作。然而，这并不意味着街上的每张 PCI VGA 卡都能在 Alpha 机器中工作。S3 Trio64、Mach64 和 Matrox Millennium 等通常可以工作。旧的基于 ET4000 的 ISA 卡对我来说也可以工作。但购买前先问问周围。

PC 世界的大多数 PCI 设备也能在 FreeBSD 基于 PCI 的机器中工作。查看 **`/sys/alpha/conf/GENERIC`** 文件了解最新信息。如果你想使用带有 PCI 桥芯片的 PCI 卡，请查看相应机器类型的讨论。在某些情况下，你可能会遇到 PCI 卡不能正确处理 PCI 奇偶校验的问题。这可能导致 panic。可以使用以下 SRM 命令禁用 PCI 奇偶校验：

```sh
>>> SET PCI_PARITY OFF
```

这不是 FreeBSD 的问题，在 Alpha 硬件上运行的所有操作系统都需要此变通方法。

如果你的系统（也）包含 EISA 扩展槽，在安装 EISA 卡或升级控制台固件后，你需要运行 EISA 配置实用程序（ECU）。

对于 Alpha CPU，你会发现多代产品。最初的 Alpha 设计是 21064。它在称为 MOS4 的芯片工艺中生产，在该工艺中制造的芯片昵称为 EV4。较新的 CPU 有 21164、21264 等。你会看到如 EV4S、EV45、EV5、EV56、EV6、EV67、EV68 等代号。双位数字的 EV 是略微改进的版本。例如，与它所基于的 EV4 相比，EV45 有改进的 FPU 和 16 kByte 片上独立 I & D 缓存。经验法则：“EV”后面的数字越高越可取（即更快/更现代）。不建议在低于 EV5 的任何东西上花钱。免费获得的硬件当然是另一回事。

对于内存，你至少需要 32 Mbyte。多年前我曾在 16 Mbyte 系统上运行 FreeBSD，但你不会喜欢那样。当我升级到 32 Mbyte 时，内核构建时间减半。请注意，SRM 控制台从总系统内存中窃取 2Mbyte（并保留它）。对于更严肃的工作，建议使用 128 Mbyte 或更多。虽然 Alpha 机器通常可以容纳大到非常大的物理内存，但 FreeBSD 限制为 1 或 2 Gbyte（取决于核心芯片组）的 RAM。此限制是由于 VM 系统的当前实现。

谈到内存时：请密切注意你的机器使用的内存类型。各种机器有非常不同的内存配置和要求。

最后一句：我预期上述内容对首次使用 Alpha 的用户来说听起来有点令人生畏。不要太被吓倒。如果阅读本文档后仍有不清楚的地方，请随时提问。

### 2.3 系统特定信息

以下是 FreeBSD 运行的硬件概述。此列表肯定会增长，查看 **`/sys/alpha/conf/GENERIC`** 可能会有所启发。

Alpha 机器通常以其项目代号最为人熟知。已知的情况下，这些代号在下面的括号中列出。

#### 2.3.1 AXPpci33（“NoName”）

NoName 是一款基于 21066 LCA（低成本 Alpha）处理器的 baby-AT 主板。NoName 最初是为 OEM 使用而设计的。LCA 芯片几乎包含了驱动 PCI 总线和内存子系统的所有逻辑。所有这些使得设计价格低廉。

由于内存接口有限，系统在缓存未命中时并不特别快。只要你留在片上缓存内，CPU 就可与 21064（第一代 Alpha）相媲美。如今获得这些主板应该非常便宜。它是一个成熟的 64 位 CPU，只是不要对速度抱有奇迹。

特性：

- 21066 Alpha CPU，166 MHz，或 21066A CPU，233MHz。也可能使用 21068 CPU，但更慢。
- 板载 Bcache / L2 缓存：0、256k 或 1 Mbyte（使用 DIL 芯片）
- PS/2 鼠标和键盘端口或 5 针 DIN 键盘（2 种主板型号）
- 内存：
  - 总线宽度：64 位
  - PS/2 型 72 针 36 位 Fast Page Mode SIMM
  - 70ns 或更好
  - 成对安装
  - 4 个 SIMM 插槽
  - 使用 ECC
- 512kB Flash ROM 用于控制台代码
- 2 个 16550A 串行端口
- 1 个并行端口
- 软盘接口
- 1 个嵌入式 IDE 接口
- 扩展：
  - 3 个 32 位 PCI 插槽（1 个与 ISA 共享）
  - 5 个 ISA 插槽（1 个与 PCI 共享）
- 使用 Symbios 53C810 芯片的嵌入式 Fast SCSI

NoName 在其 Flash ROM 中可以有 SRM *或* ARC 控制台固件。Flash ROM 不够大，无法同时容纳 ARC 和 SRM 并允许软件选择替代控制台代码。但你只需要 SRM。

NoName 的缓存是 15 或 20 ns DIL 芯片。对于 256 kByte 缓存，你可以查看废弃的 486 主板。遗憾的是，1 Mbyte 缓存的芯片比较罕见。建议至少获得 256kByte 缓存以获得性能。无缓存时它们真的很慢。

NoName 主板有一个 PC/AT 标准电源连接器。它还有一个 3.3 伏特的电源连接器。不需要急于购买新电源。3.3 伏特仅在你运行 3.3 伏特 PCI 扩展板时才需要。这些相当罕见。

IDE 接口受 FreeBSD 支持，需要在内核配置文件中添加如下行：

```ini
device ata
```

ATA 接口使用 irq 14。

遗憾的是，SRM 控制台*无法从* IDE 磁盘启动。这意味着你必须使用 SCSI 磁盘作为启动设备。

NoName 在串行控制台方面有点固执。它需要：

```sh
>>> SET CONSOLE SERIAL
```

才会使用串行控制台。像大多数其他 Alpha 型号那样拔出键盘是不够的。返回图形控制台需要：

```sh
>>> SET CONSOLE GRAPHICS
```

在串行控制台上输入。

有报告称你有时需要按 **Control**-**Alt**-**Del** 来吸引 SRM 的注意。我自己从未见过这种情况，但如果开机后你面对黑屏，这值得一试。

确保你使用真正的 36 位 SIMM，并且只使用 FPM（Fast Page Mode）DRAM。EDO DRAM 或带有假奇偶校验的 SIMM*无法工作*。主板使用 4 个额外位进行 ECC。出于同样的原因，33 位 FPM SIMM 也无法工作。

如果有选择，请购买 PS/2 变体主板。除了给你一个鼠标端口作为额外奖励外，如果你曾经想或需要运行它，它直接受 Tru64 Unix 支持。“DIN 插头”变体对于 FreeBSD 应该可以正常工作。

建议阅读 [OEM 手册](ftp://ftp.digital.com/pub/DEC/axppci/design_guide.ps)。

NoName 内核的内核配置文件必须包含：

```ini
options    DEC_AXPPCI_33
cpu EV4
```

#### 2.3.2 Personal Workstation（“Miata”）

Miata 是一款小型塔式机器，旨在放在桌下。有多种 Miata 变体。最初的 Miata 是 MX5 型号。由于它存在许多硬件设计缺陷，进行了重新设计，产生了 MiataGL。遗憾的是，从机箱外部第一眼很难区分这些变体。一个简单的检查方法是看机器背面是否有两个 USB 连接器。如果有，就是 MiataGL。MX5 型号在二手系统市场上往往更常见。

系统代号看起来像“Personal Workstation 433a”。Personal Workstation 有点拗口，通常缩写为 PWS。这意味着它有一个 433 MHz CPU，并且最初是作为 WinNT 工作站（尾部的“a”）。从第一天起就指定运行 Tru64 Unix 或 OpenVMS 的系统将带有“433au”。WinNT-Miata 可能预先配置了 IDE CDROM 驱动器。因此，系统通常命名为 PWS[433,500,600]a[u]。

还有一种带有 Kryotech 特殊 CPU 冷却系统的 Miata 型号。Kryotech 有特殊的冷却系统，并安装在 different 的机箱中。

特性：

- 21164A EV56 Alpha CPU，433、500 或 600MHz
- 21174 核心逻辑（“Pyxis”）芯片
- 板载 Bcache / L3 缓存：0、2 或 4 Mbytes（使用缓存模块）
- 内存：
  - 总线宽度：128 位宽，ECC 保护
  - 无缓冲 72 位宽 SDRAM DIMM，成对安装
  - 6 个 DIMM 插槽
  - 最大内存 1.5 GBytes
- 板载 Fast Ethernet：
  - MX5 使用 21142 或 21143 以太网芯片，取决于 PCI 转接卡版本
  - MiataGL 有 21143 芯片
  - 接口卡可以是 10/100 Mbit UTP，或 10 Mbit UTP/BNC
- 2 个板载 [E]IDE 磁盘接口，基于 CMD646（MX5）或 Cypress 82C693（MiataGL）
- 1 个 Ultra-Wide SCSI Qlogic 1040 [仅 MiataGL]
- 2 个 64 位 PCI 插槽
- 3 个 32 位 PCI 插槽（在 DEC PCI-PCI 桥芯片后面）
- 3 个 ISA 插槽（通过 Intel 82378IB PCI 到 ISA 桥芯片与 32 位 PCI 插槽物理共享）
- 2 个 16550A 串行端口
- 1 个并行端口
- PS/2 键盘和鼠标端口
- USB 接口 [仅 MiataGL]
- 基于 ESS1888 芯片的嵌入式声卡

Miata 逻辑分为两块印刷电路板。机器底部的下板有 PCI 和 ISA 插槽以及声卡等。上板有 CPU、Pyxis 芯片、内存等。请注意，MX5 和 MiataGL 使用不同的 PCI 转接板。这意味着你不能只升级到 MiataGL CPU 板（带有较新的 Pyxis 芯片），还需要不同的转接板。显然，MX5 转接板配 MiataGL CPU 板可以工作，但绝对不是受支持或经过测试的配置。其他所有内容（机箱、接线等）对于 MX5 和 MiataGL 都是相同的。

当通过 2 个 64 位 PCI 插槽进行 DMA 跨越页边界时，MX5 存在问题。32 位插槽没有这个问题，因为 PCI-PCI 桥芯片不允许有问题的传输。SRM 代码知道这个问题，如果 64 位插槽中有它不知道的 PCI 卡，它会拒绝启动系统。SRM“已知良好”的卡允许在 64 位插槽中使用。

如果你想愚弄 SRM，可以在 SRM 提示符下输入 `set pci_device_override`。只是如果你的数据神秘地被损坏，不要抱怨。

完整的命令是：

```sh
>>> SET PCI_DEVICE_OVERRIDE <vendor_id><device_id>
```

例如：

```sh
>>> SET PCI_DEVICE_OVERRIDE 88c15333
```

最激进的方法是使用：

```sh
>>> SET PCI_DEVICE_OVERRIDE -1
```

这完全禁用 PCI ID 检查，因此你可以安装任何 PCI 卡而不检查其 ID。为此，你需要相当新的 SRM 版本。

> **重要：**
>
> 此操作风险自负。

FreeBSD 内核在看到有问题的 Pyxis 芯片时会报告：

```sh
Sep 16 18:39:43 miata /kernel: cia0: Pyxis, pass 1
Sep 16 18:39:43 miata /kernel: cia0: extended capabilities: 1<BWEN>
Sep 16 18:39:43 miata /kernel: cia0: WARNING: Pyxis pass 1 DMA bug; no bets...
```

MiataGL 探测为：

```sh
Jan  3 12:22:32 miata /kernel: cia0: Pyxis, pass 1
Jan  3 12:22:32 miata /kernel: cia0: extended capabilities: 1<BWEN>
Jan  3 12:22:32 miata /kernel: pcib0: <2117x PCI host bus adapter> on cia0
```

MiataGL 没有 MX5 的 DMA 问题。使 MX5 SRM 在 64 位插槽中安装时出问题的 PCI 卡被 MiataGL SRM 毫无问题地接受。

MX5 的最新主板修订版包含该 bug 的硬件变通方法。SRM 不知道 ECO，会像以前一样抱怨未知卡。FreeBSD 内核也是如此。

Miata SRM 可以从 IDE CDROM 驱动器启动。IDE 硬盘启动已知对 MiataGL 和 MX5 磁盘都可工作，因此你可以从 IDE 磁盘引导 FreeBSD。假设驱动器合适，MX5 上的速度约为 14 Mbytes/sec。Miata 的 CMD646 芯片支持高达 WDMA2 模式，因为该芯片对于 UDMA 使用来说 bug 太多。

Miata MX5 通常使用基于 Qlogic 1040 的 SCSI 适配器。这些可由 SRM 控制台启动。请注意，Adaptec 卡*不可*由 Miata SRM 控制台启动。

MiataGL 在 PCI 转接卡上有比某些 MX5 转接卡版本更快的 PCI-PCI 桥芯片。一些 MX5 转接卡有与 MiataGL *相同*的芯片。总而言之，变化很大。

并非所有 VGA 卡都能在 PCI-PCI 桥后面工作。这表现为完全没有视频。变通方法是将 VGA 卡放在桥“之前”的 64 位 PCI 插槽中。使用 64 位插槽的图形性能通常明显更好。

MX5 和 MiataGL 都有板载声卡芯片 ESS1888。它模拟 SoundBlaster，可以通过在内核配置文件中添加以下内容来启用：

```ini
device pcm
device  sbc
```

如果你的 Miata 安装了可选缓存板，请确保它牢固就位。观察到稍微松动的缓存会导致奇怪的崩溃（显然不奇怪，但在故障排除时可能不那么明显）。缓存模块在 MX5 和 MiataGL 之间是相同的。

安装 2Mb 缓存模块可实现 10-15% 的速度提升（基于 buildworld 经过时间），但会*降低* 64 位 PCI 卡的 PCI DMA 读取带宽。在 64 位 Myrinet 卡上的基准测试结果是带宽从 149 Mbytes/sec 下降到 115 Mbytes/sec。在使用 64 位 PCI 适配器进行真正高速操作时需要记住这一点。

虽然硬件允许你安装高达 1.5Gbyte 内存，但 FreeBSD 限制为 1Gbyte，因为 DMA 代码不能正确处理 1Gbyte 以上的内存。

升级到更快的 CPU 非常简单，更换 CPU 芯片并将时钟倍频拨码开关设置为新 CPU 的速度即可。

如果你遇到如下 SRM 错误：

```sh
ERROR: scancode 0xa3 not supported on PCXAL
```

在停止 FreeBSD 后，你应该将 SRM 固件升级到 V7.2-1 或更高版本。此 SRM 版本首先在 Firmware Update CD V5.7 上提供，或在 **http://www.compaq.com/** 上提供。此 SRM 问题在 Miata MX5 和 Miata GL 上均已修复。

USB 受 FreeBSD 4.1 及更高版本支持。

拆卸机器前断开电源线，软电源开关在机器关闭时*甚至*保持部分逻辑通电。

Miata 内核的内核配置文件必须包含：

```ini
options    DEC_ST550
cpu EV5
```

#### 2.3.3 Evaluation Board 64 系列

在其推广 Alpha CPU 的尝试中，DEC 生产了许多所谓的评估板。该系列的成员有 EB64、EB64+、AlphaPC64（代号“Cabriolet”）。该系列的非 DEC 成员是 Aspen Alpine。EB64 系列评估板具有以下特性集：

- 21064 或 21064A CPU，150 至 275 MHz
- 内存：
  - 内存总线宽度：128 位
  - PS/2 型 72 针 33 位 Fast Page Mode SIMM
  - 70ns 或更好
  - 成组安装，每组 4 条
  - 8 个 SIMM 插槽
  - 使用奇偶校验内存
- Bcache / L2 缓存：0、512 kByte、1 Mbyte 或 2 Mbytes
- 21072（“APECS”）芯片组
- Intel 82378ZB PCI 到 ISA 桥芯片（“Saturn”）
- 双 16550A 串行端口
- 并行打印机端口
- Symbios 53C810 Fast-SCSI（AlphaPC64 上无）
- IDE 接口（仅 AlphaPC64 上有）
- 嵌入式 10 Mbit 以太网（AlphaPC64 上无）
- 2 个 PCI 插槽（AlphaPC64 上 4 个插槽）
- 3 个 ISA 插槽

Aspen Alpine 略有不同，但与 EB64+ 足够接近，可以运行 EB64+ SRM EPROM（我的可以..）。Aspen Alpine 没有嵌入式以太网，有 3 个而不是 2 个 PCI 插槽。它带有 2 Mbytes 缓存，已焊在主板上。它有跳线可以选择使用 60、70 或 80ns SIMM 速度。

36 位 SIMM 工作正常，3 位简单地保持未使用。请注意，系统使用 Fast Page Mode 内存，不是 EDO 内存。

EB64+ SRM 控制台代码 housed 在 UV 可擦除 EPROM 中。EB64+ 没有简单的 flash SRM 升级。EB64+ 可用的最新 SRM 版本反正相当古老。

EB64+ SRM 可以启动 53C810 和 Qlogic1040 SCSI 适配器。Qlogic 的陷阱是 SRM 下载到 Qlogic 芯片上的固件非常旧。EB64+ SRM 没有可用更新。所以你也困在旧的 Qlogic 代码中。当我想在 Alpine 上使用 Qlogic 的 Ultra-SCSI 驱动器时，我遇到了相当多的问题。FreeBSD 内核可以编译包含更新的 Qlogic 固件修订版。这不是默认的，因为它给内核增加了数百 kByte 的膨胀。在 FreeBSD 4.1 及更高版本中，isp 固件包含在内核可加载模块中。所有这些可能意味着你需要使用非 Qlogic 适配器来启动。

AlphaPC64 板通常带有 ARC 控制台固件。SRM 控制台代码可以从软盘加载到 Flash ROM 中。

AlphaPC64 的 IDE 接口不能从 SRM 控制台启动。启用它需要在内核配置文件中添加以下行：

```ini
device ata
```

ATA 接口使用 irq 14。

请注意，这些板需要一个为 CPU 提供 3.3 伏特的电源。

对于 EB64 系列机器，内核配置文件必须包含：

```ini
options    DEC_EB64PLUS
cpu EV4
```

#### 2.3.4 Evaluation Board 164（“EB164、PC164、PC164LX、PC164SX”）系列

EB164 是较新的设计评估板，基于 21164A CPU。此设计已用于“衍生”多种变体，其中一些被 OEM 制造商/组装商使用。Samsung 自行设计了 PC164LX，只有 32 位 PCI，而 Digital 变体有 64 位 PCI。

- 21164A，多种速度变体 [EB164、PC164、PC164LX]
- 21164PC [仅 PC164SX]
- 21171（Alcor）芯片组 [EB164]
- 21172（Alcor2）芯片组 [PC164]
- 21174（Pyxis）芯片 [164LX、164SX]
- Bcache / L3 缓存：EB164 使用特殊缓存 SIMM
- 内存总线：128 位 / 256 位
- 内存：
  - PS/2 型 SIMM，成组安装，每组 4 或 8 条
  - 36 位，Fast Page Mode，使用 ECC [EB164 / PC164]
  - SDRAM DIMM，成组安装，每组 2 条，可使用非 ECC 或 ECC [PC164SX]
  - SDRAM DIMM，成组安装，每组 2 条，使用 ECC [PC164LX]
- 2 个 16550A 串行端口
- PS/2 型键盘和鼠标
- 软盘控制器
- 并行端口
- 32 位 PCI
- 64 位 PCI [部分型号]
- 通过 Intel 82378ZB PCI 到 ISA 桥芯片的 ISA 插槽

使用 8 条 SIMM 进行 256 位宽内存可以比 4 条 SIMM/128 位宽内存产生有趣的速度提升。显然，所有 8 条 SIMM 必须是相同类型才能使其工作。系统必须明确设置为使用 8 条 SIMM 内存排列。你必须有 8 条 SIMM，分布在 2 个 bank 中的 4 条 SIMM 无法工作。对于 AlphaPC164，你最多可以有 1Gbyte 的 RAM，使用 8 条 128Mbyte SIMM。手册指示最大为 512 Mbyte。

SRM 可以从 Qlogic 10xx 板或 Symbios 53C810[A] 启动。Symbios 810 的较新修订版（如 Symbios 810AE）不被 PC164 上的 SRM 识别。PC164 SRM 似乎不识别基于 Symbios 53C895 的主机适配器（用 Tekram DC-390U2W 测试）。另一方面，有报告称某无品牌 Symbios 53C985 板可以工作。像 Tekram DC-390F（基于 Symbios875）这样的卡已确认在 PC164 上工作正常。遗憾的是，这似乎取决于芯片/板的实际版本。

Symbios 53C825[a] 也可作为启动适配器。Diamond FirePort 虽然基于 Symbios 芯片，但不能由 PC164SX SRM 启动。据报道 PC164SX 可以用基于 Symbios825、Symbios875、Symbios895 和 Symbios876 的卡正常启动。此外，据报道 Adaptec 2940U 和 2940UW 可用于启动（在 SRM V5.7-1 上验证）。Adaptec 2930U2 和 2940U2[W] 无法工作。

带有 SRM 固件版本 5.8 或更高版本的 164LX 和 164SX 可以从 Adaptec 2940 系列适配器启动。然而，使用 Adaptec 3940UW 的测试表明该型号不受支持。164SX SRM 识别 Intel 8255x 以太网卡，显示为 eia。使用这样的 NIC 允许网络启动。

总结：就 SCSI 适配器而言，该系列机器被“赐予”了具有挑战性的兼容性。

在 164SX 上，你最多可以有 1 Gbyte 的 RAM。据报道 4 条普通（PC100 或 PC133）256MB DIMM 工作正常。512MB DIMM 是否也能工作目前未知。你可以使用 ECC 或非 ECC DIMM。非 ECC 的与 PC 中常见的相同。遗憾的是，164SX 对它喜欢的 DIMM 相当挑剔，所以准备好测试和实验。

PCI 桥芯片有时不被 164SX 接受，在这些情况下它们会导致 SRM 错误和内核 panic。这似乎取决于卡是否被 SRM 控制台识别并因此正确初始化。164SX 的板载 IDE 接口相当慢，Promise 卡可提供 3-4 倍的速度提升。

在 PC164 上，SRM 有时会丢失其变量设置。“对于 PC164，当前的迷信说法是，为了避免丢失设置，你想先降级到 SRM 4.x，然后升级到 5.x。”观察到的一个错误示例是：

```sh
ERROR: ISA table corrupt!
```

降级到 SRM4.9，然后执行：

```sh
>>> ISACFG -INIT
```

接着执行：

```sh
>>> INIT
```

使问题消失。一些 PC164 所有者报告他们从未见过此问题。

在 PC164SX 上，AlphaBIOS 允许你选择在下一次开机时使用 SRM 作为控制台。此选择似乎没有任何效果。换句话说，无论你选择什么，都会得到 AlphaBIOS。解决方法是用 PC164SX 的 SRM 代码重新刷新控制台 ROM。这将覆盖 AlphaBIOS 并让你获得所需的 SRM 控制台。SRM 代码可在 Compaq 网站上找到。

164LX 的 flash ROM 中可以有 SRM 控制台代码或 AlphaBIOS 代码，因为 flash ROM 太小，无法同时容纳两者。

如果你的 SRM 版本足够新，PC164 可以从 IDE 磁盘启动。

EB164 需要一个提供 3.3 伏特的电源。PC164 不实现 ATX 电源所需的 PS_ON 信号。一个简单的开关将此信号拉低到地线，允许你使用标准 ATX 电源。

对于 EB164 类机器，内核配置文件必须包含：

```ini
options         DEC_EB164
cpu             EV5
```

#### 2.3.5 AlphaStation 200（“Mustang”）和 400（“Avanti”）系列

Digital AlphaStation 200 和 400 系列系统是早期的低端基于 PCI 的工作站。200 和 250 系列是桌面机箱，400 系列是桌边小型塔式机箱。

特性：

- 21064 或 21064A CPU，速度从 166 到 333 MHz
- DECchip 21071-AA 核心逻辑芯片组
- Bcache / L2 缓存：512 Kbytes（200 和 400 系列）或 2048KBytes（250 系列）
- 内存：
  - 64 位总线宽度
  - 8 到 384 MBytes RAM
  - 70 ns 或更好的 Fast Page DRAM
  - 分三对安装（200 和 400 系列）
  - 分两组，每组四条（250 系列）
  - 内存子系统使用奇偶校验
- PS/2 键盘和鼠标端口
- 两个 16550 串行端口
- 并行端口
- 软盘接口
- 32 位 PCI 扩展插槽（AS400 系列 3 个，AS200 和 250 系列 2 个）
- ISA 扩展插槽（AS400 系列 4 个，AS200 和 250 系列 2 个）（部分 ISA/PCI 插槽物理共享）
- 嵌入式基于 21040 的以太网（200 和 250 系列）
- 嵌入式 Symbios 53c810 Fast SCSI-2 芯片
- Intel 82378IB（“Saturn”）PCI-ISA 桥芯片
- 图形为嵌入式 TGA 或 PCI VGA（取决于型号）
- 16 位声卡（200 和 250 系列）

系统使用奇偶校验内存 SIMM，但这些不需要 36 位宽的 SIMM。33 位宽的 SIMM 足够，36 位 SIMM 也可接受。EDO 或 32 位 SIMM 无法工作。支持 4、8、16、32 和 64 Mbyte SIMM。

假设你的内核配置文件中有以下行，据报道 AS200 和 AS250 声音硬件工作正常：

```ini
device pcm
```

声音设备使用端口 0x530、IRQ 10 和 drq 0。你还需要在 **`device.hints`** 文件中指定 flags 0x10011。

AlphaStation 200 和 250 系列有自动 SCSI 终结器。这意味着只要你将电缆插入外部 SCSI 连接器，系统的内部终结器就会被禁用。这也意味着你不应将未终结的电缆留在机器中。

AlphaStation 400 系列有一个控制终结的 SRM 变量。如果你连接了外部 SCSI 设备，必须使用以下命令设置此 SRM 变量：

```sh
>>> SET CONTROL_SCSI_TERM EXTERNAL
```

如果只有内部 SCSI 设备，请使用：

```sh
>>> SET CONTROL_SCSI_TERM INTERNAL
```

对于 AlphaStation-[24][05]00 机器，内核配置文件必须包含：

```ini
options    DEC_2100_A50
cpu EV4
```

#### 2.3.6 AlphaStation 500 和 600（“Alcor”和“Maverick”用于 EV5，“Bret”用于 EV56）

AS500 和 600 曾经是高端 EV5 / PCI 工作站。EV6 机器已经取代它们成为前沿产品。AS500 是深蓝色机箱（TopGun blue）的台式机，AS600 是坚固的桌边机箱。AS600 有一个漂亮的 LCD 面板，用于观察 SRM 启动的早期阶段。

特性：

- 21164 EV5 CPU，266、300、333、366、400、433、466 或 500 MHz（AS500）或 266、300 或 333 MHz（AS600）
- 21171（Alcor）或 21172（Alcor2）核心逻辑芯片组
- 缓存：
  - 2 或 4 Mb L3 / Bcache（AS600 at 266 MHz）
  - 4 Mb L3 / Bcache（AS600 at 300 MHz）
  - 2 或 8 Mb L3 / Bcache（8 Mb 仅 500 MHz 版本）
  - 2 至 16 Mb L3 / Bcache（AS600；3 个缓存 SIMM 插槽）
- 内存总线宽度：256 位
- AS500 内存：
  - 业界标准 72 位宽缓冲 Fast Page Mode DIMM
  - 8 个 DIMM 插槽
  - 成组安装，每组 4 条
  - 最大内存 1 GB（333 MHz CPU 上最大 512 Mb）
  - 使用 ECC
- AS600 内存：
  - 业界标准 36 位 Fast Page Mode SIMM
  - 32 个 SIMM 插槽
  - 成组安装，每组 8 条
  - 最大内存 1 GB
  - 使用 ECC
- 基于 Qlogic 1020 的宽 SCSI 总线（AS500 1 条总线/芯片，AS600 2 条总线/芯片）
- 基于 21040 的 10 Mbit 以太网适配器，同时有 Thinwire 和 UTP 连接器
- 扩展：
  - AS500：
    - 3 个 32 位 PCI 插槽
    - 1 个 64 位 PCI 插槽
  - AS600：
    - 2 个 32 位 PCI 插槽
    - 3 个 64 位 PCI 插槽
    - 1 个 PCI/EISA 物理共享插槽
    - 3 个 EISA 插槽
    - 1 个 PCI 和 1 个 EISA 插槽默认被占用
- 21050 PCI-to-PCI 桥芯片
- Intel 82375EB PCI-EISA 桥（仅 AS600）
- 2 个 16550A 串行端口
- 1 个并行端口
- 16 位音频 Windows Sound System，在专用插槽中（AS500），在 EISA 插槽中（AS600，这是 ISA 卡）
- PS/2 键盘和鼠标端口

早期机器有 Fast SCSI 接口，后期是 Ultra SCSI。AS500 将其单一 SCSI 总线与内部和外部设备共享。对于 Fast SCSI 总线，你被限制在机箱外 1.8 米总线长度。AS500 Qlogic ISP1020A 芯片可以通过设置 SRM 变量以 Ultra 模式运行。然而，FreeBSD 遵循 Qlogic 芯片勘误并将总线速度限制为 Fast。

当心 AS500 上的古老 SRM 版本。当你看到 FreeBSD 报告奇怪 SCSI 速度时，如：

```sh
cd0 at isp0 bus 0 target 4 lun 0
cd0: <DEC RRD45   DEC 0436> Removable CD-ROM SCSI-2 device
cd0: 250.000MB/s transfers (250.000MHz, offset 12)
```

是时候升级 SRM 控制台固件了。

AS600 有一个 Qlogic SCSI 芯片专用于内部设备，另一个 Qlogic SCSI 芯片专用于外部 SCSI 设备。将卡（除了两个 SCSI 接口外还容纳以太网芯片）保持在 DEC 推荐的最顶部 32 位 PCI 插槽中。如果你将其插入 64 位插槽之一，会出现机器检查。

在 AS500 中，DIMM 成组安装，每组 4 条，采用“物理交错”布局。因此，一组 4 条 DIMM *不是* 4 条物理相邻的 DIMM。请注意，DIMM *不是* SDRAM DIMM。

在 AS600 中，内存 SIMM 放置在两块内存子卡上。SIMM 成组安装，每组 8 条。两块内存子卡必须相同配置。

请注意，AS500 和 AS600 都是 EISA 机器。这意味着在添加 EISA 卡或更改板载 I/O 的配置设置后，你必须从软盘运行 EISA 配置实用程序（ECU）。对于没有物理 EISA 插槽的 AS500，ECU 用于配置板载声音接口等。

AS500 板载声音可以通过在内核配置文件中添加如下行来使用：

```ini
device pcm
```

使用 ECU，我将 AS500 配置为使用 IRQ 10、端口 0x530、drq 0。相应的条目连同 flags 0x10011 必须放入 **`device.hints`** 文件。请注意，flags 值相当非标准。

AS600 的 PCI 插槽有一个特殊之处。AS600（或者更确切地说是包含 SCSI 适配器的 PCI 扩展卡）不允许 I/O 端口映射，因此其后面的所有设备必须使用内存映射。如果你在使 Qlogic SCSI 适配器工作时遇到问题，请在 **`/boot/loader.rc`** 中添加以下选项：

```ini
set isp_mem_map=0xff
```

在引导安装内核之前，可能需要在引导加载程序提示符下输入此命令。

对于 AlphaStation-[56]00 机器，内核配置文件必须包含：

```ini
options    DEC_KN20AA
cpu EV5
```

#### 2.3.7 AlphaServer 1000（“Mikasa”）、1000A（“Noritake”）和 800（“Corelle”）

AlphaServer 1000 和 800 系列机器旨在作为部门服务器。它们在机箱和主板/CPU 方面有相当多的变体。一般来说，有基于 21064（EV4）CPU 的机器和基于 21164（EV5）的机器。CPU 在子卡上，CPU 类型（EV4 或 EV5）必须与使用的主板匹配。

AlphaServer 800 有一个更小的迷你塔式机箱，它缺少 StorageWorks SCSI 热插拔机箱。AS1000 和 AS1000A 之间的主要区别是 AS1000A 有 7 个 PCI 插槽，而 AS1000 只有 3 个 PCI 插槽，并有 EISA 插槽。

带有 EV5/400 MHz CPU 的 AS800 后来重新品牌化为“DIGITAL Server 3300[R]”，带有 EV5/500 MHz CPU 的 AS800 后来重新品牌化为“DIGITAL Server 3305[R]”。

特性：

- 21064 EV4[5] CPU，200、233 或 266 MHz；21164 EV5[6] CPU，300、333 或 400 MHz（或 500 MHz，仅 AS800）
- 内存：
  - 总线宽度：128 位，带 ECC
  - AS1000[A]：
    - 72 针 36 位 Fast Page Mode SIMM，70ns 或更好
    - 16（EV5 机器）或 20（EV4 机器）个 SIMM 插槽
    - 最大内存 1 GB
    - 使用 ECC
  - AS800：使用 60ns 3.3 伏特 EDO DIMM
- 嵌入式 VGA（部分主板型号）
- 3 个 PCI、2 个 EISA、1 个 64 位 PCI/EISA 组合（AS800）
- 7 个 PCI、2 个 EISA（AS1000A）
- 2 个 PCI、1 个 EISA/PCI、7 个 EISA（AS1000）
- 基于 Symbios 810 [AS1000] 或 Qlogic 1020 [AS1000A] 的嵌入式 SCSI

AS1000 机器有多种机箱类型。落地式、机架式、有或无 StorageWorks SCSI 机箱等。电子设备相同。

AS1000 系统：所有基于 EV4 的机器使用标准 PS/2 型 36 位 72 针 SIMM，每组 5 条。第五条 SIMM 用于 ECC。所有基于 EV5 的机器使用标准 PS/2 型 36 位 72 针 SIMM，每组 4 条。ECC 基于 SIMM 的 4 个额外位（36 位中的 4 位）完成。EV5 主板有 16 个 SIMM 插槽，EV4 主板有 20 个插槽。

```

在串行控制台上输入。

对于 AS800，你想检查 Ultra-Wide SCSI 是否确实处于 Ultra 模式。这可以使用控制台固件升级 CDROM 上的 **`EEROMCFG.EXE`** 实用程序完成。

cpu EV5     # 取决于安装的 CPU 型号
```

#### 2.3.11 AlphaServer 2000（“DemiSable”）、2100（“Sable”）、2100A（“Lynx”）

AlphaServer 2[01]00 机器旨在作为部门服务器。这是中型机器。它们是多 CPU 机器，最多可安装 2 个 CPU（AS2000）或 4 个 CPU（2100[A]）。有落地式和 19" 机架式机箱。机架式变体有不同数量的 I/O 扩展插槽、不同的最大 CPU 数和不同的最大内存大小。一些机箱带有整体 StorageWorks 架以容纳热插拔 SCSI 磁盘。曾经有一个升级计划，可以通过更换 I/O 背板（C-bus 背板保持不变）将你的 Sable 机器转换为 Lynx。也有 CPU 升级可用。

- 21064 EV4[5] CPU，200、233、275 MHz；或 21164 EV5[6] CPU，250、300、375、400 MHz
- 缓存：随 CPU 型号变化；每个 CPU 1、4 或 8Mbyte
- 嵌入式软盘控制器，驱动 2.88 Mbytes 驱动器
- 嵌入式 10Mbit 21040 以太网 [仅 AS2100]
- 2 个串行端口
- 1 个并行端口
- PS/2 型键盘和鼠标端口

规格为 200 MHz 的 CPU 实际运行在 190 MHz。CPU 最大数量为 4。所有 CPU 必须是相同类型/速度。

如果任何处理器曾被标记为故障，即使它们已被更换（或重新插拔），它们仍将保持故障标记，直到你在 SRM 控制台上发出命令：

```sh
>>> CLEAR_ERROR ALL
```

并循环启动机器。对于其他模块（IO 和内存）可能也是如此，但尚未验证。

机器使用专用内存板。这些板位于与 CPU 板共享的 128 位 C-bus 上。DemiSable 最多支持 1GB，Sable 最多 2GB。其中一个内存总线插槽可以容纳 CPU 或内存卡。4 CPU 机器最多可以有 2 块内存板。

一些内存板型号容纳 SIMM。这些被称为 SIMM 载体。也有焊接内存芯片而不是 SIMM 的内存模块。这些被称为“扁平内存模块”。

SIMM 板以八条一组使用 72 针 36 位 FPM 内存，70ns 或更快。支持的 SIMM 类型有 1M x36 bit（4 Mbyte）、2M x36bit（8 Mbyte）和 4M x36 bit（16 Mbyte）。每块内存板可容纳 4 个 SIMM bank。单块内存板上不能混合 SIMM 尺寸。第一块内存模块必须填满 SIMM 后才能开始填充下一块内存模块。请注意插槽之间的间距并不大，所以确保你的 SIMM 物理上合适（在购买之前..）。

Lynx 和 Sable 在串行控制台方面有点固执。它们需要：

```sh
>>> SET CONSOLE SERIAL
```

才会使用串行控制台。像许多其他 Alpha 型号那样拔出键盘是不够的。返回图形控制台需要：

```sh
>>> SET CONSOLE GRAPHICS
```

在串行控制台上输入。在 Lynx 上，将 VGA 卡保留在主 PCI 插槽之一。EISA VGA 卡对插槽不敏感。

机器配备小型 OCP（操作员控制面板）LCD 屏幕。在系统初始化期间，此屏幕上显示自检消息。你可以使用 SRM 在那里放置你自己的小文本：

```sh
>>> SET OCP_TEXT "FreeBSD"
```

SRM 命令：

```sh
>>> SHOW FRU
```

生成配置概述，包括模块序列号、硬件修订版和错误日志计数。

Sable、DemiSable 和 Lynx 都有基于 Symbios 810 的板载 Fast SCSI。检查它是否设置为 Fast SCSI 速度：

```sh
>>> SHOW PKA0_FAST
```

设置为 1 时，它正在协商 Fast 速度。

```sh
>>> SET PKA0_FAST 1
```

启用 Fast SCSI 速度。

AS2100[A] 配备 StorageWorks 7 插槽 SCSI 笼。机柜内可以添加第二个笼。AS2000 有单个 7 插槽 SCSI 笼，无法扩展额外的笼。请注意，这些笼中的插槽位置映射到 SCSI ID 的方式与标准 StorageWorks 架不同。使用单总线配置时，从上到下的插槽 ID 为 0、4、1、5、2、6、3。

笼也可以设置为提供两条独立的 SCSI 总线。这用于嵌入式 RAID 控制器，如 KZPSC（Mylex DAC960）。分离总线的插槽 ID 分配，从上到下：0A、0B、1A、1B、2A、2B、3A、3B。其中 A 和 B 表示一条 SCSI 总线。在单总线配置中，SCSI 笼背面的终结器模块位于顶部。跳线模块位于底部。对于分离总线操作，这两个模块反转。可以通过注意终结器上的芯片来区分终结器和跳线。跳线上没有任何有源组件。

DemiSable 有 7 个 EISA 插槽和 3 个 PCI 插槽。Sable 有 8 个 EISA 和 3 个 PCI 插槽。Lynx 较新，有 8 个 PCI 和 3 个 EISA 插槽。Lynx PCI 插槽分组为 4 组。最靠近 CPU/内存插槽的 4 个 PCI 插槽是主插槽，即在 PCI 桥芯片之前。请注意，与预期相反，主 PCI 插槽是编号最高的（PCI4 - PCI7）。

在 EISA 插槽中添加/更改扩展卡或升级控制台固件后，确保运行 EISA 配置实用程序（从软盘）。这通过插入 ECU 软盘并输入以下命令完成：

```sh
>>> RUNECU
```

> **注意：**
>
> EISA 插槽目前不受支持，但 Compaq Qvision EISA VGA 适配器被视为 ISA 设备。因此它作为控制台工作正常。

计划用于 C-bus 的特殊扩展 I/O 模块。它们是否曾经面世尚不清楚。无论如何，FreeBSD 从未与 ExtIO 模块一起验证过。

机器可以配备冗余电源。请注意，机箱配备了联锁开关，当机箱打开时会切断电源。系统的冷却风扇是速度控制的。当机器有超过 2 个 CPU 和超过 1 块内存板时，必须使用双电源。

```ini

cpu EV5 #取决于安装的 CPU 型号
```

Rawhide 可以配备多种 CPU 模块。CPU 模块有无外部缓存版本。在所有情况下，安装的 CPU 模块必须始终是相同速度。NT-only 和完整 Tru64/VMS CPU 的混合工作正常。然而，这将导致系统向操作系统报告自己为 Digital Server 730x（即 NT-only 变体）。FreeBSD 不在乎，但这样的系统不允许运行 Tru64 或 VMS。

Rawhide 最多使用 8 个 RAM 模块。这些模块成对使用，向总线提供 72 位（包括 ECC 位）。内存可以是 EDO RAM 或同步 DRAM。完全填充的 AS4100 有 4 对内存模块。AS4000 型号限制为 2 对内存模块。如果有选择，使用 SDRAM 以获得最佳性能。最高容量的内存板必须在标记为 MEM0L 和 MEM0H 的内存插槽中。允许混合内存板尺寸。EDO 和 SDRAM 的混合也可以工作（假设你不在单个模块对中混合 EDO 和 SDRAM）。EDO 和 SDRAM 的混合会导致*整个*内存子系统以较慢的 EDO
