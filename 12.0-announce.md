# FreeBSD 12.0-RELEASE 发行公告

日期：2018 年 12 月 11 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 12.0-RELEASE 现已发布。 这是 stable/12 分支的首个版本。

部分亮点如下：

* OpenSSL 已更新至版本 1.1.1a (LTS)。
* Unbound 已更新至版本 1.8.1，并默认启用 DANE-TA。
* OpenSSH 已更新至版本 7.8p1。
* 为 [sshd(8)](https://man.freebsd.org/cgi/man.cgi?query=sshd&sektion=8&format=html) 添加了额外的 [capsicum(4)](https://man.freebsd.org/cgi/man.cgi?query=capsicum&sektion=4&format=html) 支持。
* Clang、LLM、LLD、LLDB、compiler-rt 和 libc++ 已更新至版本 6.0.1。
* [vt(4)](https://man.freebsd.org/cgi/man.cgi?query=vt&sektion=4&format=html) Terminus BSD 控制台字体已更新至版本 4.46。
* [bsdinstall(8)](https://man.freebsd.org/cgi/man.cgi?query=bsdinstall&sektion=8&format=html) 工具现在支持 UEFI+GELI 作为安装选项。
* **VIMAGE** 内核配置选项已默认启用。
* **NUMA** 选项已默认启用，适用于 amd64 **GENERIC** 和 **MINIMAL** 内核配置。
* 新增 [netdump(4)](https://man.freebsd.org/cgi/man.cgi?query=netdump&sektion=4&format=html) 驱动，通过它可以在系统崩溃后将内核崩溃转储传输到远程主机。
* [vt(4)](https://man.freebsd.org/cgi/man.cgi?query=vt&sektion=4&format=html) 驱动带来了性能改进，文本绘制速度提高了 2 到 6 倍。
* 对当前一代硬件的图形支持进行了多项改进。
* 默认启用了 armv6 和 armv7 上的 [capsicum(4)](https://man.freebsd.org/cgi/man.cgi?query=capsicum&sektion=4&format=html) 支持。
* UFS/FFS 文件系统已更新，合并了 TRIM/`BIO_DELETE` 命令，由于同时发送的 TRIM 消息减少，从而减少了读写请求。
* NFS 版本 4.1 服务器已更新，包含 pNFS 服务器支持。
* [pf(4)](https://man.freebsd.org/cgi/man.cgi?query=pf&sektion=4&format=html) 包过滤器现在可以在使用 [vnet(9)](https://man.freebsd.org/cgi/man.cgi?query=vnet&sektion=9&format=html) 的 [jail(8)](https://man.freebsd.org/cgi/man.cgi?query=jail&sektion=8&format=html) 中使用。
* [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 工具已更新，增加了 NVMe 设备仿真。
* [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 工具现在可以在 [jail(8)](https://man.freebsd.org/cgi/man.cgi?query=jail&sektion=8&format=html) 中运行。
* 多项 Lua [loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 改进。
* KDE 已更新至版本 5.12.5。
* 以及更多……

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

## 获取途径

FreeBSD 12.0-RELEASE 现已支持 amd64、i386、powerpc、powerpc64、powerpcspe、sparc64、armv6、armv7 和 aarch64 架构。

FreeBSD 12.0-RELEASE 可从可启动 ISO 镜像或通过网络安装。 部分架构还支持从 USB 闪存盘安装。 所需文件可按以下说明下载。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/12.0R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/12.0R/announce.asc>

发行版所提供镜像的用途如下：

dvd1::
包含安装 FreeBSD 基础操作系统、文档和少量预构建 package（旨在启动图形工作站）所需的全部内容。
还支持启动到基于「livefs」的救援模式。
若你能刻录并使用 DVD 介质，这应是你所需的全部。

disc1::
包含 FreeBSD 基础操作系统。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

bootonly::
支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。
从 CD 启动后，需要执行基于网络的安装（例如从 FTP 服务器）。

memstick::
可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-12.0-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

务必确保目标（`of=`）正确。

mini-memstick::
可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-12.0-RELEASE-amd64-mini-memstick.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

FreeBSD/arm SD 卡镜像::
可写入 SD 卡并用于启动受支持的 arm 系统。
SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 [ssh(1)](https://man.freebsd.org/cgi/man.cgi?query=ssh&sektion=1&format=html) 访问。
此外，`root` 用户密码设置为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 [dd(1)](https://man.freebsd.org/cgi/man.cgi?query=dd&sektion=1&format=html) 工具，将 _KERNEL_ 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-12.0-RELEASE-arm-armv6-KERNEL.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。
